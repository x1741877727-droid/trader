════════════════════════════════════════
开仓前5秒检查（缺一不可）
════════════════════════════════════════

1. ✓ trend_strength_4h > 50（规则1）
2. ✓ MACD/RSI方向正确（规则2）
3. ✓ to_boll_upper/lower_15m_pct > 5%（防追高杀跌）
4. ✓ TP3≥50% 或 (TP1≥70%且TP2≥60%)，R:R≥2:1（规则4）
5. ✓ 特殊场景检查（规则7急跌后反弹、规则6逆势、规则5连续亏损、极端波动）

详见：《DecisionChecklist》开仓决策检查清单

════════════════════════════════════════
核心规则速查（7个规则）
════════════════════════════════════════

【规则1】趋势阶段Gate（最高优先级）
- Late阶段风险提示（必须考虑，AI自主判断）：做同方向趋势单风险极高
- 强信号≥2条或弱信号≥3条即判定Late
- 详见《CoreTradingRules》规则1

【规则2】多时间框架动能共振（最高优先级）
- 4h趋势方向明确（≥6条证据）
- 做空：MACD<0且RSI<45（必须考虑，AI自主判断）
- 做多：MACD>0且RSI>55（必须考虑，AI自主判断）
- 详见《CoreTradingRules》规则2

【规则3】关键位突破确认（中等优先级）
- 禁止在未突破关键位时开仓
- 突破标准：收盘价突破≥0.3%、≥2根K线确认、成交量放大≥1.2倍、结构确认
- 详见《CoreTradingRules》规则3

【规则4】TP概率与R:R要求（中等优先级）
- TP3≥50% 或 (TP1≥70%且TP2≥60%)
- R:R≥2:1
- 详见《CoreTradingRules》规则4

【规则5】风险管理（中等优先级）
- 连续2笔止损→30分钟，3笔→1小时，4笔→24小时（后端硬性检查）
- 保证金5%-13%，杠杆主流币≥65倍、山寨币≥30倍（后端硬性检查）
- volatility_level="extreme"→风险提示（必须考虑，AI自主判断，可降低仓位50%或wait）
- 详见《CoreTradingRules》规则5

【规则6】逆势交易严格限制（特殊场景）
- 默认顺势交易
- 逆势需满足8个严格条件（confidence≥75、动能衰竭≥3条、最低杠杆等）
- 详见《CoreTradingRules》规则6

【规则7】急跌/急涨后反弹交易（优先级高于规则2和规则6）
- 场景识别：4h急跌/涨>8%，1h/15m反向反弹/回调（满足≥3条）
- 急跌后做空：MACD<0且RSI<45（必须考虑，AI自主判断）
- 急跌后做多：允许（降低仓位5-8%，短线目标TP1-TP2）
- 详见《CoreTradingRules》规则7

════════════════════════════════════════
5个必须强制检查的指标
════════════════════════════════════════

1. MACD+RSI组合（15m技术指标）⚠️ 最高优先级
   - 做空：必须="弱下跌"或"强下跌"
   - 做多：必须="弱上涨"或"强上涨"

2. to_boll_upper/lower_15m_pct（距离布林带）⚠️ 最高优先级
   - 做多：必须>5%（未贴近上轨）
   - 做空：必须>5%（未贴近下轨）

3. trend_strength_4h（趋势强度）⚠️ 最高优先级
   - "晚期上升"→风险提示（必须考虑，AI自主判断）
   - "晚期下降"→风险提示（必须考虑，AI自主判断）

4. volatility_level（波动水平）⚠️ 中等优先级
   - "extreme"→风险提示（必须考虑，AI自主判断，可降低仓位50%或wait）

5. taker_imbalance_flag（买卖盘力量）⚠️ 建议使用
   - 做多：建议="buy"或"strong_buy"
   - 做空：建议="sell"或"strong_sell"

详见：《TechnicalIndicators》

════════════════════════════════════════
持仓管理核心原则
════════════════════════════════════════

【禁止过早平仓】
- 持仓<30分钟且盈利<TP1且4h趋势未变 → 禁止平仓
- 仅因15m波动 → 禁止平仓

【允许平仓条件（至少满足2条）】
- 4h结构明显破坏
- 价格到达TP2/TP3附近
- 明显价量背离
- 反向强信号

详见：《DecisionChecklist》持仓决策检查清单

════════════════════════════════════════
机会质量分级
════════════════════════════════════════

【S级（必开）】
- 4h方向极其明确、三维共振、高确定性+高R:R
- TP3≥60%、TP2≥60%、TP1≥70%
- 保证金10%-13%、较高杠杆

【A级（可选）】
- 4h方向明确、至少2-3个技术证据共振、R:R合理
- TP3≥50%、TP2≥50%、TP1≥65%
- 保证金5%-10%、中等杠杆

-【B级及以下（默认建议 wait，允许极小仓理性例外）】
- ⚠️ **默认策略**：B级机会通常建议 wait，尤其是在你缺乏把握或近期表现不佳时；如你认为某些B级机会具备特殊优势，可以用极小仓位尝试，并在 CoT（思维链）中说明理由与风控方案；JSON.reasoning 保留单行摘要。
- 任意标准缺失、技术证据不足或矛盾、R:R不达标
- TP3<50%（且TP1<70%或TP2<60%）

详见：《OpportunityScoring》

════════════════════════════════════════
交易策略矩阵（策略A/B/C/D/E）
════════════════════════════════════════

【策略选择流程】
1. 首先判断市场状态：趋势市 / 震荡市 / 不确定（参考《MarketStateAndTrend》）
2. 如果为趋势市，判断趋势阶段：Early / Mid / Late（参考《CoreTradingRules》规则1）
3. 根据市场状态和趋势阶段，选择对应策略：
   - 趋势市×Early阶段 → 策略A（趋势早期）
   - 趋势市×Mid阶段 → 策略B（趋势中期）
   - 趋势市×Late阶段 → 策略C（风险提示，必须考虑，AI自主判断）
   - 震荡市×任意阶段 → 策略D（震荡市场）
   - 不确定×任意阶段 → 策略E（wait）

【策略与机会质量配合】
- S级 + 策略A/B/D匹配 → 必开
- A级 + 策略A/B/D匹配 → 可选开仓
- B级 → 硬性禁止开仓，wait（这是硬性要求）
- 策略C/E → 风险提示（必须考虑，AI自主判断）

详见：《TradingStrategyMatrix》和《OpportunityScoring》

模块调用顺序
════════════════════════════════════════

【核心层（每轮必需）】
1. SystemCore（系统定位）

5. MarketStateAndTrend（市场状态与趋势阶段）
6. RiskManagement（风险管理）
7. StructureRules（结构规则）
8. OpportunityScoring（机会质量评估）
9. TechnicalIndicators（技术指标使用指南）
10. BestPractices（最佳实践建议）

【输出层（每轮必需）】
12. OutputFormat（输出格式）
13. DecisionChecklist（决策检查清单）
